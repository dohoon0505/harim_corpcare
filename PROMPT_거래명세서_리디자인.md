# Claude Code 작업 프롬프트 — 거래명세서 조회 페이지 리디자인 적용

아래 프롬프트를 그대로 Claude Code에 전달하세요.

---

## 목표

`#/app/invoice` 거래명세서 조회 페이지(`js/pages/invoice.js` + `css/pages/invoice.css`)를 **`mockups/invoice/01-simple.html` 시안과 동일한 UX/비주얼**로 재구현한다.
시안이 유일한 기준(source of truth)이다. 참고 캡쳐: `mockups/invoice/captures/01-simple.png`.

## 절대 제약

1. **A4 명세서 문서는 절대 변경 금지.** `js/invoice-doc.js`(794×1123 인라인 스타일 렌더러)와 `printInvoiceDoc()`을 그대로 사용한다. 문서 내부 디자인·규격·마크업을 손대지 않는다. 공개 링크 페이지(`/invoice/`, `js/public-invoice.js`)도 변경하지 않는다.
2. **순수 HTML/CSS/바닐라 JS 유지.** 프레임워크·빌드도구·의존성 추가 금지. `mount(root, { nav }) → cleanup 반환` 규약, `dom.js`의 `html\`\``/`on()` 헬퍼 사용.
3. **hex는 `css/tokens.css`에만.** 주문 리디자인(PROMPT_경조상품주문_리디자인.md)에서 추가한 토스 팔레트 토큰(`--c-order-blue` 등)을 그대로 재사용한다. 아직 없다면 그 프롬프트의 규칙대로 tokens.css에 추가한다.
4. **앱 셸(사이드바·헤더) 유지.** 페이지 콘텐츠 영역만 변경. `components.css`·`shell.css` 수정 금지.
5. **데스크톱 전용.** 기존 기능(PDF 다운로드, 공개 링크 복사, 기간 변경, 계산서 발급 동의)은 전부 유지한다.

## 화면 구조 (시안 그대로)

페이지 타이틀 "거래명세서" + 서브카피 "월별 거래 내역을 확인하고 명세서와 계산서를 발급받을 수 있어요" 아래 2단 레이아웃:

- **좌측 — A4 문서 미리보기**: `invoiceDoc()` 결과를 얇은 보더 + 소프트 섀도의 프레임(`border-radius:4px`)에 담아 표시. 폭 794px 고정.
- **우측 — 레일(sticky, 324px)**, 위에서부터:
  1. **조회 기간 카드**: 연도·월 **커스텀 드롭다운 2개** (기존 `openPeriodModal` 모달을 대체). 주문 리디자인에서 `ui.js`에 추가한 makeDropdown 헬퍼를 재사용하되, 이 페이지에서는 패널이 **아래로** 전개된다. 연도 2024~2026, 월 01~12.
  2. **요약 카드**: "{선택월} 결제금액" + 금액(큰 글씨), 결제·정산 대금기한(레드), 계산서 발급 상태 칩(동의 필요=레드 틴트 / 동의 완료=그린 틴트).
  3. **액션 버튼**: `{YYYY}_{MM} 거래명세서 다운로드`(블루 솔리드 → `printInvoiceDoc` 호출), `공개 링크 복사 · 로그인 없이 열람`(그레이 고스트 → 기존 `issueLink`/`publicInvoiceUrl` 로직 유지, 복사 성공은 토스트로 피드백).
  4. **계산서 발급 동의 카드**: 안내 문구 + 동의 버튼. 동의 시 카드가 그린 완료 상태(체크 + 동의 일자)로 전환되고 요약 카드 칩도 "동의 완료"로 변경. 기존 `openAgreementModal`의 확인 스텝을 유지하고 싶다면 `simpleModal` 확인 후 완료 상태로 전환해도 좋다(시안은 즉시 전환).

기간 변경 시 문서(제목·귀속·발행일·거래내역·정산금액)와 요약 카드가 함께 갱신된다. 데이터 없는 월은 문서 거래내역에 "해당 월의 거래 내역이 없습니다" 행을 표시한다(문서 규격은 유지).

## 커스텀 드롭다운 · 토스트

- 드롭다운 스펙은 주문 리디자인 프롬프트와 동일(50px 트리거, 12px 라운드 패널, 선택 행 블루+흰 글자, 열릴 때 선택값 중앙 스크롤, 바깥 클릭·ESC 닫힘, aria 속성). **공용 헬퍼로 두고 두 페이지가 공유**한다.
- 토스트: 화면 하단 중앙, 다크 필 형태, 2초 후 사라짐. 공개 링크 복사·동의 완료 피드백에 사용.

## 데이터

기존 `INVOICE_DATA`를 월별 맵 구조로 확장한다(데모: 2026-03, 2026-04). 문서 데이터 스키마(`title/period/buyer/supplier/items/account/total`)는 invoice-doc.js 계약 그대로 유지. 합계는 items에서 계산해 표시한다.

## 수용 기준

1. `node serve.mjs` → `#/app/invoice`에서 시안과 동일한 레이아웃·톤 렌더링, A4 문서는 기존과 픽셀 동일.
2. 연/월 드롭다운으로 기간 변경 시 문서·요약 동시 갱신, 데이터 없는 월 빈 상태 표시. 기존 기간 변경 모달은 제거.
3. PDF 다운로드가 기존 `printInvoiceDoc` 경로로 동작(인쇄 결과물 변화 없음).
4. 공개 링크 복사가 기존 토큰 발급 로직으로 동작하고 토스트 피드백 표시.
5. 계산서 발급 동의 → 카드 완료 상태 + 요약 칩 "동의 완료" 전환, 재진입 시 상태 유지(기존 state 저장 방식 따름).
6. 다른 라우트·공개 명세서 페이지에 회귀 없음. 콘솔 에러 없음. cleanup 시 document 레벨 리스너 해제.

## 진행 방식

1. `mockups/invoice/01-simple.html`을 정독해 마크업·스타일·동작을 이식 기준으로 삼는다.
2. (필요 시) tokens.css 토큰 확인 → `css/pages/invoice.css` 재작성 → `js/pages/invoice.js` 재작성(invoice-doc.js 호출부 유지) 순서로 진행.
3. 단계마다 `node serve.mjs`로 수동 확인, 특히 PDF 인쇄와 공개 링크를 실제로 열어 검증한다.
