## 목표

`#/app` 경조상품 주문 페이지(`js/pages/order.js` + `css/pages/order.css`)를 **`mockups/order/08-brand.html` 시안과 동일한 UX/비주얼**로 재구현한다. 시안이 유일한 기준(source of truth)이다 — 작업 전 시안 파일을 정독하고 마크업·CSS·JS 로직을 그대로 이식 기준으로 삼는다.

## 절대 제약

1. **순수 HTML/CSS/바닐라 JS 유지.** 프레임워크·빌드도구·의존성 추가 금지. `mount(root, { nav }) → cleanup 반환` 규약, `dom.js`의 `html\`\``/`on()` 헬퍼, `icons.js` 사용.
2. **hex는 `css/tokens.css`에만.** 시안은 이미 기존 토큰 값으로 만들어져 있다. 시안에서 쓰였지만 토큰에 없는 값만 승격 추가한다:
   `--c-blue-sel:#f8f9ff`(선택 행 워시 — 기존 order.css 하드코딩 값의 토큰화) · `--c-dark-hover:#2b2b33` · `--c-orange-hover:#d94e24`. 그 외 신규 색 추가 금지, 외부 서비스 팔레트 금지.
3. **앱 셸(사이드바·헤더) 유지.** 페이지 콘텐츠만 교체. `components.css`·`shell.css` 수정 금지(공용 헬퍼는 `ui.js`에 추가 허용).
4. **데스크톱 전용.** 접수 동작은 기존과 동일한 데모 수준(성공 화면 전환)으로 유지 — 서버 연동·주문 저장 로직 신설 금지.
5. 기존 담당자 게이트 화면과 모달 4종(리본/보내는분/간편접수/완료)은 제거하고 시안 구조로 대체한다.

## 페이지 구조 (시안 그대로)

앱 캔버스(`--c-bg`) 위에 기존 `pageTitle` 패턴의 헤더(🌸 경조화환 주문), 그 아래 **플로팅 화이트 시트**(12px 라운드 · 헤어라인 보더 · 미세 그림자)가 주문 영역 전체를 감싼다. 시트 상단은 "경조상품 간편주문" 타이틀 + 우측 "경조화환을 간편하게 주문해보세요!" 보조 카피, 이어서 **언더라인 탭 4단계**(경조사 및 상품 선택 → 배송지 정보 입력 → 리본문구 입력 → 작성내용 확인). 탭 활성 = 오렌지 언더라인, 완료 단계 = 오렌지 점, 도달한 탭만 클릭 이동.

### STEP 1 — 경조사 및 상품 선택 (단계적 공개)

1. **경조사 배너 2개 가로 배치** (harim_mob `intake-card` 패턴 그대로): 근조·부고(촛불 사진) | 축하·행사(웨딩 사진, `background-size:130%; position:60% 70%`). 88px, 좌→우 다크 그라데이션(`rgba(8,11,20,.82)→.05`), 흰 라벨 16/600 + 서브 12, 우측 흰 셰브론, hover 그림자, active `scale(.98)`. 선택 시 오렌지 링(outline 2px) + 오렌지 체크 서클. 기타 경조사 없음.
2. 배너 선택 시 **reveal 애니메이션**(fade + translateY, .38s)으로 링크 박스 공개. 제목·플레이스홀더는 경조사별로 전환:
   근조 → "부고장 링크가 있다면 붙여넣어주세요!" / 축하 → "청첩장 또는 초대장 링크가 있다면 붙여넣어주세요!". **불러오기 = 오렌지 강조 버튼**, 아래 **"링크 없이 직접 작성하기 →" 보조 버튼**(화이트+보더).
3. 불러오기 성공(기존 `MOCK_URL_DB` 재사용, 성공 시 배송지·받는분 자동입력) 또는 직접작성 클릭 시 같은 애니메이션으로 **상품 선택** 공개: 헤어라인 라디오 리스트, 해당 경조사 3단화환 2종, 선택 = 연블루 워시(`--c-blue-sel`) + 블루 라디오/금액.

### STEP 2 — 배송지 정보 입력

- 주소(자동입력 시 그린 "링크에서 자동입력" 배지, 수동 수정 시 해제) / 받는분 성함 / 연락처(자동 하이픈). 필드 포커스 = 인디고 보더 + `--ring-field`.
- **배송 일시**: 세그먼트(날짜·시간 지정 ↔ 즉시배송) +
  - **커스텀 데이트피커**: 드롭다운과 동일한 트리거/패널 디자인, 라벨 "2026. 07. 03 (금)". 달력 패널(월 헤더 ‹ › · 요일 · 날짜 그리드), **오늘부터 최대 30일까지만 선택 가능**(범위 밖 비활성, 월 이동 화살표도 범위 제한), 선택일 = 블루 솔리드, 오늘 = 블루 아웃라인, 하단 "오늘부터 최대 30일까지 예약할 수 있어요".
  - **시·분 커스텀 드롭다운**(harim mob 스타일, 선택 행 블루 솔리드): 시 09~18시, **18시 선택 시 분 00·10·20·30만** 노출되고 기존 40/50분은 30분으로 자동 보정.
  - **즉시배송 = 영업시간(09:00~18:30) 규정**: 시간 외 세그먼트 비활성 + 안내 문구, 1분 주기 재판정, 선택 시 "4시간 이내(HH:MM 전) 배송" 안내.
  - 안내: "배송 시간은 09:00 ~ 18:30 사이에서 지정할 수 있어요 · 오전 11시 이전 접수 건은 당일 배송할 수 있어요".
- 드롭다운·데이트피커 동작 공통 규칙: 한 번에 하나만 열림, 바깥 클릭/ESC 닫힘, 열릴 때 선택값 중앙 스크롤(시·분), aria(listbox/dialog, expanded, selected). **`ui.js`에 `makeDropdown()`·`makeDatepicker()` 공용 헬퍼로 구현**(페이지 재구현 금지 — 거래명세서 등 다른 페이지가 재사용 예정).

### STEP 3 — 리본문구 입력

문구 입력(20자 제한 + 우측 카운터) + 경조사별 추천 칩(기존 `COMMON_PHRASES` 재사용, 선택 = 블루 소프트) + "접수 후에는 문구를 수정할 수 없어요" 레드 경고. 보내는분 = 헤어라인 라디오 리스트(`store.get().profiles`), "＋ 새 명의 등록하기" 블루 링크(`#/app/profile`).

### STEP 4 — 작성내용 확인

확인 리스트(경조사·상품/배송지/배송 일시/리본 문구, 각 행 블루 "변경" → 해당 단계 점프) + **결제 금액 오렌지 잉크 26px** + 옵션 박스(주문 담당자 select = `store.get().contacts`, 배송완료 알림 칩 3종 — **ON 시 블루 + 체크 SVG 표시**). CTA "주문 접수하기" = 오렌지. 접수 시 완료 화면(그린 체크 서클, 접수 요약 박스, "새 주문 작성하기" + "주문내역 보기"→`#/app/orders`).

각 단계 하단: "이전"(화이트+보더) / "다음"(오렌지, 해당 단계 유효성 충족 전 비활성).

## 수용 기준

1. `node serve.mjs` → `#/app`이 시안 08-brand와 시각적으로 동일하게 렌더(셸 안, 회색 캔버스 + 화이트 시트).
2. 배너 선택 → 링크 박스(경조사별 문구) → 상품 공개의 단계적 공개가 애니메이션과 함께 동작. 기타 경조사 없음.
3. `kakao.com` 포함 URL 불러오기 → 근조 자동 선택 + 주소/받는분 자동입력 + 그린 배지 + 상품 공개. 미지원 URL → 인라인 에러.
4. 데이트피커: 오늘~+30일만 선택 가능(총 31일), 월 이동 제한, 선택일 블루 표시, 라벨 "YYYY. MM. DD (요일)".
5. 시 드롭다운 09~18시, 18시 선택 시 분 00~30 제한 + 50분→30분 자동 보정. 캘린더·시·분 패널은 상호 배타적으로 열림.
6. (시스템 시각 변경으로) 영업시간 외 즉시배송 비활성 + 안내 노출 확인.
7. 알림 칩 ON/OFF 시 체크 SVG 표시/숨김. Step4 "변경" 점프, 탭 완료 점·이동 규칙 동작.
8. 접수 → 완료 화면, "새 주문 작성하기" → 초기화된 Step 1.
9. 다른 라우트 회귀 없음 · 콘솔 에러 없음 · cleanup 시 document 레벨 리스너와 `setInterval`(영업시간 재판정) 해제.
10. `css/pages/order.css`에 raw hex 0건(전부 `var()` 참조).

1. `mockups/order/08-brand.html` 정독 → 2. `tokens.css` 토큰 3개 추가 + 에셋 복사 → 3. `ui.js`에 makeDropdown/makeDatepicker → 4. `css/pages/order.css` 재작성 → 5. `js/pages/order.js` 재작성 → 각 단계마다 `node serve.mjs`로 수동 확인.
