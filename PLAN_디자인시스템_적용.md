# HDS(올해의경조사 디자인 시스템) 적용 상세 계획

기준 문서: `mockups/design-system.html` (라이브 스타일가이드 — 브라우저에서 열어 확인)

---

## 1. 배경과 문제 진단

| 문제 | 내용 |
|------|------|
| 톤앤매너 분열 | 주문(07)·거래명세서(01) 시안이 외부 서비스(토스) 팔레트(#3182f6 블루, #191f28 잉크, #f2f4f6 그레이)로 제작되어, 오렌지+잉크 기반의 로그인·회원가입·기존 앱과 브랜드가 단절됨 |
| 법적 리스크 | 타사의 디자인 값·색상 체계를 그대로 차용하는 것은 트레이드 드레스 관점의 리스크가 있어 제거 필요 |
| 시스템 부재 | 페이지마다 버튼·인풋·드롭다운 스타일이 제각각 정의되어 유지보수 비용 증가 |

**해결 방향**: 로그인·회원가입 페이지의 기존 브랜드 아이덴티티(브랜드 오렌지 `#f15a2a` · 잉크 `#111118` · 소프트 그레이)를 유일한 원천으로 삼아 자체 팔레트 램프와 컴포넌트 시스템(HDS)을 정의하고, 모든 화면을 여기에 정렬한다. 외부 팔레트 값은 전량 치환한다.

## 2. 디자인 원칙 (전 화면 공통)

1. **오렌지는 행동, 블루는 정보** — 화면당 오렌지 솔리드 CTA 1개. 링크·토글·정보 배지만 블루(`#4169e1`). 나머지는 뉴트럴.
2. **잉크는 무게, 그레이는 구조** — 잉크(`#111118`)는 다크 패널·토스트·다크 버튼에만. 위계는 색이 아닌 보더·여백·타이포로.
3. **무드는 상황에만** — 근조 `#3c3c48` / 축하 `#b01254`는 경조 상황 구분 전용. 일반 UI 침범 금지.

## 3. 토큰 체계 개편 — `css/tokens.css`

원칙: **기존 변수는 삭제하지 않고 유지**(전 페이지 회귀 방지), HDS 램프를 추가하고 기존 변수를 별칭으로 연결한다.

### 3-1. 신규 추가 (HDS 램프)

```css
/* ── HDS 브랜드 오렌지 램프 ── */
--hds-orange-50:#fff1ec;  --hds-orange-100:#ffdfd2; --hds-orange-200:#ffbfa4;
--hds-orange-300:#fb9a72; --hds-orange-400:#f4713f; --hds-orange-500:#f15a2a;
--hds-orange-600:#d94000; --hds-orange-700:#b23300;
/* ── 잉크 ── */
--hds-ink-900:#111118; --hds-ink-700:#2b2b33; --hds-ink-500:#4b4b55;
/* ── 뉴트럴 ── */
--hds-gray-25:#f9f9f9; --hds-gray-50:#f5f5f7; --hds-gray-100:#ececef;
--hds-gray-200:#e0e0e4; --hds-gray-300:#d0d0d6; --hds-gray-400:#a9a9b2;
--hds-gray-500:#83838d; --hds-gray-600:#5d5d66;
/* ── 무드 (order.css의 하드코딩 hex를 토큰으로 승격) ── */
--hds-obit:#3c3c48; --hds-obit-hover:#2e2e38; --hds-obit-soft:#ececf1;
--hds-wed:#b01254;  --hds-wed-hover:#94104a;  --hds-wed-soft:#fbe9f1;
/* ── 라디우스 추가 ── */
--r-tile:14px;
```

### 3-2. 기존 변수 매핑 (변경 없음 — 값이 이미 램프와 일치)

`--c-orange = orange-500`, `--c-orange-ink = orange-600`, `--c-orange-soft = orange-50`, `--c-dark = ink-900`, `--c-bg = gray-25`, `--c-surface-2 = gray-50`, 시맨틱(`--c-success/danger/warn` 계열)과 블루(`--c-blue`) 전부 유지.

### 3-3. 외부 팔레트 → HDS 치환 표 (시안 수정용)

| 용도 | 제거할 값 (토스) | HDS 대체 |
|------|------------------|----------|
| 주 CTA · 활성 | `#3182f6` / `#1b64da` | `--hds-orange-500` / `--hds-orange-600` |
| 포커스 링 · 선택 워시 | `#e8f3ff` / `#f0f7ff` | `--hds-orange-50` + `--ring` |
| 강조 텍스트 · 다크 | `#191f28` / `#333d4b` | `--hds-ink-900` / `--c-text-2` |
| 보조 텍스트 | `#4e5968` `#6b7684` `#8b95a1` | `--c-text-3` / `--c-text-muted` / `--c-text-faint` |
| 보더 · 필 | `#e5e8eb` / `#f2f4f6` / `#f9fafb` | `--hds-gray-200` / `--hds-gray-50` / `--hds-gray-25` |
| 성공 그린 | `#03b26c` / `#e6f6ef` | `--c-success-ink` / `--c-success-bg` |
| 오류 레드 | `#f04452` / `#fdeef0` | `--c-danger-ink` / `--c-danger-bg` |
| 토글 | 블루 유지 | `--c-blue` (기존 앱 관례) |

## 4. 컴포넌트 시스템 — `css/components.css`에 `hds-` 클래스 추가

스타일가이드에 정의된 스펙 그대로 추가한다(기존 클래스는 손대지 않음).

| 컴포넌트 | 클래스 | 핵심 스펙 |
|----------|--------|-----------|
| 버튼 | `.hds-btn` + `--primary/--dark/--secondary/--ghost`, 사이즈 `--md/--sm` | 높이 48/42/34 · 라디우스 10 · primary는 화면당 1개 |
| 텍스트 필드 | `.hds-field` + `.hds-input`, 상태 `.is-error/.is-success` | 높이 50 · 라디우스 10 · 포커스 = 오렌지 보더+링 |
| 커스텀 드롭다운 | `.hds-dd` (트리거/패널/옵션) | 선택 행 = 오렌지 워시+오렌지 잉크+체크 · 바깥클릭/ESC 닫힘 · aria listbox · JS는 `ui.js`의 `makeDropdown()` 공용 헬퍼 |
| 세그먼트 | `.hds-seg` | 그레이 트랙 + 화이트 선택 필 |
| 토글 | `.hds-toggle` | on = 블루 (기존 관례 유지) |
| 체크/라디오 | `.hds-check` / `.hds-radio` | on = 오렌지 |
| 칩 | `.hds-chip` | 선택 = 오렌지 워시+오렌지 잉크 |
| 상태 배지 | `.hds-badge--success/danger/warn/info/neutral/obit/wed` | 시맨틱 워시 + 잉크 텍스트 |
| 카드 | `.hds-card` (+`__head/__body`) | 라디우스 14 · 헤어라인 보더 · sh-card |
| 스텝퍼 | `.hds-steps/.hds-step` | 회원가입 위저드와 동일 패턴 (완료=워시, 활성=솔리드) |
| 테이블 | `.hds-table` | 헤더 그레이 필 · 행 헤어라인 |
| 토스트 | `.hds-toast` | 잉크 필 · 하단 중앙 · 2초 |
| 빈 상태 | `.hds-empty` | 아이콘 서클 + 안내문 |

## 5. 적용 로드맵

### Phase 0 — 기반 (반나절 · 회귀 위험 없음)
- `tokens.css`에 3-1 토큰 추가, `components.css`에 4장 클래스 추가 (추가만, 수정 없음).
- `ui.js`에 `makeDropdown()` 공용 헬퍼 추가 (스타일가이드 데모 구현 이식).
- 검증: 전 라우트 렌더 변화 없음 확인.

### Phase 1 — 시안 재스킨 (외부 팔레트 제거)
- `mockups/order/07-simple.html`, `mockups/invoice/01-simple.html`의 `:root` 토큰 블록을 HDS 값으로 교체(3-3 치환 표 적용). 레이아웃·플로우·인터랙션은 유지.
- 드롭다운 선택 행: 블루 솔리드 → 오렌지 워시 스타일로 변경.
- A4 명세서 문서(`invoice-doc` 규격)는 원래 브랜드 오렌지였으므로 변경 없음.
- 검수 기준: **로그인 페이지와 나란히 놓고 봤을 때 같은 서비스로 보일 것.**

### Phase 2 — 주문 페이지 실적용
- `PROMPT_경조상품주문_리디자인.md`로 진행하되, "토스 팔레트 토큰 추가" 지시를 **본 계획 3장(HDS 토큰)으로 대체**하고 Phase 0 산출물을 사용.
- `css/pages/order.css`는 `hds-` 공용 클래스 + 페이지 전용 클래스 최소화로 재작성.

### Phase 3 — 거래명세서 실적용
- `PROMPT_거래명세서_리디자인.md` 동일 방식. `invoice-doc.js` 불변 원칙 유지.

### Phase 4 — 잔여 화면 정렬 + 가드레일
- 실시간 주문내역·정산·프로필·상품안내 페이지의 버튼/필드/배지를 `hds-` 클래스로 점진 치환.
- 가드레일: ① 신규 hex는 tokens.css 외 금지(코드리뷰 체크) ② 화면당 primary 버튼 1개 ③ 새 컴포넌트는 스타일가이드에 먼저 추가 후 사용 ④ `mockups/design-system.html`을 살아있는 기준으로 유지.

## 6. 검증 체크리스트 (각 Phase 공통)
